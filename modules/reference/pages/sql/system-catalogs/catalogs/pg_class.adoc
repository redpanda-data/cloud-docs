= pg_class
:description: The pg_class stores information about tables and indexes in the database.

The `pg_class` stores information about tables and indexes in the database. It contains exactly one row per table (or index) created in the database. It mimics the https://www.postgresql.org/docs/15/catalog-pg-class.html[pg_class] PostgreSQL system catalog.

== Columns

[NOTE]
====
This table is designed for compatibility with tools that require PostgreSQL system tables, so it mostly has dummy data. Please note that not all columns in `pg_class` are applicable to every type of relation.
====
The following columns are available for querying in `pg_class`:

[width="100%",cols="15%,5%,80%",options="header",]
|===
|Column |Type |Description
|`oid` |`int` |This column represents the table/index object ID (OID) generated by oxla
|`relname` |`text` |This column represents the table/index name as specified by the user during creation
|`relnamespace` |`int` |This column represents the OID of the namespace the relation resides in
|`relhasindex` |`bool` |Returns `true` if the table has any indexes
|`relkind` |`text` |This column represents the type of relation: `r` for tables and `i` for indexes
|`reltype` |`int` |_Unused_
|`reloftype` |`int` |_Unused_
|`relowner` |`int` |_Unused_
|`relam` |`int` |_Unused_

|`reltuples` |`float` |_Unused_
|`relallvisible` |`int` |_Unused_
|`reltoastrelid` |`int` |_Unused_
|`relisshared` |`bool` |_Unused_
|`relpersistence` |`text` |_Unused_
|`relnatts` |`int` |_Unused_
|`relchecks` |`int` |_Unused_
|`relhasrules` |`bool` |_Unused_

== Example

[arabic]
. Create a table and define its schema:

[source,sql]
----
CREATE TABLE customer_orders (
    order_id INT,
    customer_id INT,
    order_date DATE,
    total_amount INT
);
----

[arabic, start=2]
. Create an index on the `customer_orders` table for the `customer_id` column:

[source,sql]
----
CREATE INDEX idx_customer_id ON customer_orders (customer_id);
----

[arabic, start=3]
. Query the `pg_class` catalog to retrieve information about the `customer_orders` table and the index you’ve just created:

[source,sql]
----
SELECT oid, relname, relkind, relhasindex, relnamespace
FROM pg_class
WHERE relname IN ('customer_orders', 'idx_customer_id');
----

[arabic, start=4]
. The query will return information about the `customer_orders` table and the index:

[source,sql]
----
 oid  |     relname     | relkind | relhasindex | relnamespace
------+-----------------+---------+-------------+--------------
 1013 | idx_customer_id | i       | f           |            0
 1012 | customer_orders | r       | t           |            0
(2 rows)
----

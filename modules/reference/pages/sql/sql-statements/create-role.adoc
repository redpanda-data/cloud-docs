= CREATE ROLE
:description: The CREATE ROLE statement creates a new role (user) in Redpanda SQL. Only a superuser can create roles.
:page-topic-type: reference

The `CREATE ROLE` statement creates a new role in Redpanda SQL. `CREATE USER` is a synonym for `CREATE ROLE`. Only a superuser can run this statement.

A password is required when creating a role.

== Syntax

[source,sql]
----
CREATE { ROLE | USER } role_name [WITH] PASSWORD 'password' [SUPERUSER | NOSUPERUSER];
----

* `role_name`: Name of the new role. Cannot be `CURRENT_USER` or `CURRENT_ROLE`.
* `PASSWORD '_password_'`: Required. The password must be a non-empty string literal.
* `SUPERUSER`: Optional. Grants superuser privileges to the new role. Superusers bypass authorization checks.
* `NOSUPERUSER`: Optional. Explicitly marks the role as a non-superuser. This is the default if neither `SUPERUSER` nor `NOSUPERUSER` is specified.
* `WITH`: Optional. Has no effect; provided for PostgreSQL compatibility.

== Examples

Create a regular user with a password:

[source,sql]
----
CREATE ROLE alice WITH PASSWORD 'secret123';
----

Create a superuser:

[source,sql]
----
CREATE USER admin_user WITH PASSWORD 'admin_secret' SUPERUSER;
----

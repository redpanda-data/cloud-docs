= Overview
:description: String functions are used to analyze and manipulate string values.

String functions are used to analyze and manipulate string values. Oxla supports the following string related functions and operators:

==== String Functions

[width="100%",cols="20%,80%",options="header",]
|===
|*Function* |*Description*
|xref:sql:reference/sql-functions/string-functions/length.adoc[LENGTH()] |Returns the number of characters in a string
|xref:sql:reference/sql-functions/string-functions/lower.adoc[LOWER()] |Makes string lowercase
|xref:sql:reference/sql-functions/string-functions/upper.adoc[UPPER()] |Makes string upper case
|xref:sql:reference/sql-functions/string-functions/starts-with.adoc[STARTS++_++WITH()] |Checks if a string starts with a specified substring
|xref:sql:reference/sql-functions/string-functions/ends-with.adoc[ENDS++_++WITH()] |Checks if a string ends with a specified substring
|xref:sql:reference/sql-functions/string-functions/concat.adoc[CONCAT()] |Adds two or more strings together
|xref:sql:reference/sql-functions/string-functions/substr.adoc[SUBSTR()] |Extracts a substring from a string
|xref:sql:reference/sql-functions/string-functions/strpos.adoc[STRPOS()] |Finds the position at which the substring starts within the string
|xref:sql:reference/sql-functions/string-functions/regex/regexp-match.adoc[REGEXP++_++MATCH()] |Matches a POSIX regular expression pattern to a string
|xref:sql:reference/sql-functions/string-functions/regex/regexp-replace.adoc[REGEXP++_++REPLACE()] |Substitutes new text for substrings that match POSIX regular expression patterns
|xref:sql:reference/sql-functions/string-functions/replace.adoc[REPLACE()] |Finds and replace occurences of a substring in a string
|xref:sql:reference/sql-functions/string-functions/position.adoc[POSITION()] |Returns the position of the first occurrence of a substring in a string
|===

==== String Operators

[width="100%",cols="51%,49%",options="header",]
|===
|*Operator* |*Description*
|text ~ text -++>++ boolean |Returns `true` if the first argument matches the pattern of the second argument in case sensitive match
|text ~++*++ text -++>++ boolean |Returns `true` if the first argument matches the pattern of the second argument in a case-insensitive match
|text !~ text -++>++ boolean |Returns `true` if the first argument does not match the pattern of the second argument in case sensitive match.
|text !~++*++ text -++>++ boolean |Returns `true` if the first argument does not match the pattern of the second argument in a case-insensitive match
|===
